# Apigee X for Spring Boot Developers

A from-the-basics **Apigee X** learning site for experienced **Java / Spring Boot**
developers. Every Apigee concept is bridged to a Spring concept you already know,
taught against a **UK Open Banking / FAPI 1.0 Advanced** running domain, with
copy-pasteable code in every session, hands-on labs in a free Apigee X evaluation
org, and an **Apolaki** (Filipino sun/war deity) dark theme.

**Live site:** https://sreenivas-sadhu-prabhakara.github.io/apigeex-for-spring-devs/

## Course shape

**33 atomic, cumulative sessions across 7 parts.** Each session is self-contained —
one objective, an explicit *builds-on* link, a **Spring Boot bridge** (and where the
analogy breaks), a hands-on lab, a *verify it* check, common failure modes, and a
stretch goal.

1. **Foundations & the Apigee mental model** — what a gateway *product* is.
2. **Flow & mediation engine** — the request/response flow model and the policy toolkit.
3. **Security, identity & API products** — threats, the entitlement chain, OAuth, JWT, mTLS.
4. **FAPI 1.0 Advanced & the OB trust framework** — the OB ecosystem and the FAPI profile.
5. **Building the UK Open Banking APIs** — DCR, consent, AISP, PISP, CoF, end to end.
6. **Operations, delivery & observability** — environments, CI/CD, analytics, productisation.
7. **Capstone** — assemble a production-grade Open Banking platform and a go-live review.

## Repository layout

```
content/
  curriculum.json     Single source of truth: parts → sessions (metadata, builds_on, bridge, stretch)
  index.md            Landing page
  session-01.md …     One Markdown file per session (01–33)
assets/
  style.css           Apolaki design system (dark)
  app.js              Copy buttons, progress, reading bar, TOC spy
  widgets.js/.css     Interactive widgets (pipeline, sequence, statemachine, quiz, …)
  svg/                Hand-authored teaching diagrams (Apolaki-themed)
  heroes/             Part-opener hero art + manifest.json + PROMPTS.md
apolaki_pygments.py   Custom Pygments style (solar-on-charcoal)
build.py              Renders content/ → static HTML in docs/
tools/                Helper scripts (hero placeholder generator)
docs/                 Pre-rendered site served by GitHub Pages (committed)
```

## Build

```bash
pip install -r requirements.txt
python3 build.py            # renders content/ → docs/
```

Then open `docs/index.html`, or serve locally:

```bash
python3 -m http.server -d docs 8000   # http://localhost:8000  (widgets need HTTP)
```

The site is **pre-rendered** — `docs/` is committed and served by GitHub Pages
(`main` → `/docs`) with `.nojekyll`. No CI build runs on push; regenerate locally
with `python3 build.py` whenever you edit `content/` or `assets/`.

## Authoring a session

Each `content/session-NN.md` starts with `# <code> — <title>` and uses Apolaki
callout components (Markdown admonitions), rendered by `build.py`:

```markdown
!!! bottomline "Bottom line"
    The one thing you'll be able to do.

!!! bridge "Spring Boot bridge"
    The Spring concept this maps onto.

!!! breaks "Where the analogy breaks"
    Where the mapping stops holding.

!!! lab "Lab — …"
!!! verify "Verify it"
!!! failure "Common failure modes"
!!! stretch "Stretch goal"
```

Crumbs, the builds-on band, the hero (on part openers), the "mark complete"
control, and prev/next are generated by `build.py` from `curriculum.json` — don't
hand-write them. Fenced ` ```mermaid `, ` ```widget `, and highlighted code blocks
are supported.

## Hero images

`assets/heroes/` ships abstract SVG placeholders (`tools/gen_placeholders.py`). To
use generated raster art, follow `assets/heroes/PROMPTS.md`, drop the PNGs in, and
point the matching `src` in `assets/heroes/manifest.json` at them.
